All-Pairs Shortest Paths: Shortest Path MM и Repeated Squaring

1
Точна дефиниция — по лекции на проф. Марков

APShP (All-Pairs Shortest Paths, Изч. Задача 30): Дадени са ориентиран граф G = (V, E) и тегловна функция w. Намери δ(u, v) за всички двойки u, v ∈ V.

Характеризация (рекурсивна декомпозиция): Нека L(m)[i][j] = теглото на най-лек път от i до j, използващ най-много m ребра. Тогава:

  • L(1)[i][j] = w(i,j) ако (i,j) ∈ E, иначе ∞ (за i ≠ j); L(1)[i][i] = 0
  • L(m)[i][j] = mink{ L(m-1)[i][k] + w(k,j) } — разширяване с едно ребро
  • При липса на отрицателни цикли: L(n-1)[i][j] = δ(i,j)
Алгоритъм 37: Shortest-Path-MM
-- Алгоритъм 37: Shortest-Path-MM (матрично умножение)
Shortest-Path-MM(W: матрица на теглата, n: брой върхове)
1  L ← W              -- L = L^(1): един ред на матрицата на теглата
2  for m ← 2 to n-1
3      L' ← нова матрица n×n
4      for i ← 1 to n
5          for j ← 1 to n
6              L'[i][j] ← ∞
7              for k ← 1 to n
8                  L'[i][j] ← min(L'[i][j], L[i][k] + W[k][j])
9      L ← L'
10 return L           -- L = L^(n-1) = матрицата на δ(i,j)

Сложност по време: Θ(n³ · n) = Θ(n⁴) — n-2 итерации × Θ(n³) за едно "умножение".

Алгоритъм 38: Repeated Squaring (Повторно повдигане на квадрат)
-- Алгоритъм 38: Repeated Squaring — ускоряване до Θ(n³·lg n)
Repeated-Squaring(W: матрица на теглата, n: брой върхове)
1  L ← W              -- L = L^(1)
2  m ← 1
3  while m < n-1 do
4      L ← Extend(L, L)   -- L^(2m) = L^(m) "умножено" по себе си
5      m ← 2m
6  return L

-- Обосновка: вместо n-2 умножения правим само ⌈lg(n-1)⌉ умножения
-- след ⌈lg(n-1)⌉ стъпки m ≥ n-1 → L = L^(m) = L^(n-1) = δ матрица
Пример — Repeated Squaring за n=5
n=5, трябва L^(4): Стъпка 1: m=1, L=L^(1) → L = L^(1)⊗L^(1) = L^(2), m=2 Стъпка 2: m=2, L=L^(2) → L = L^(2)⊗L^(2) = L^(4), m=4 m=4 ≥ n-1=4 → СПИРАМЕ Само 2 умножения вместо 3 (за m=2,3,4)! За n=100: ⌈lg(99)⌉ = 7 умножения вместо 98!
Ключови неща за запомняне
  • Аналогия с матрично умножение: "+" → "min", "×" → "+"
  • Shortest-Path-MM: Θ(n⁴); Repeated Squaring: Θ(n³·lg n)
  • И двата са по-бавни от Floyd-Warshall Θ(n³) — само теоретичен интерес
  • Обосновка на горната граница: след ⌈lg(n-1)⌉ удвоявания, m ≥ n-1

Floyd-Warshall (Алгоритъм 39)

2
Точна дефиниция — Характеризация

Характеризация на най-кратък път (Floyd-Warshall): Нека D(k)[i][j] = теглото на най-лек път от i до j, чиито вътрешни върхове принадлежат на подмножеството {1, 2, ..., k}.

Рекурсивна декомпозиция:

  • D(0)[i][j] = w(i,j) ако (i,j) ∈ E; 0 ако i=j; ∞ иначе
  • D(k)[i][j] = min( D(k-1)[i][j], D(k-1)[i][k] + D(k-1)[k][j] )
  • D(n)[i][j] = δ(i,j) — финалният отговор

Интуиция: или най-лекият път от i до j не минава през k (първи член), или минава — и тогава се разбива на i→k и k→j (втори член).

Алгоритъм 39: Floyd-Warshall
-- Алгоритъм 39: Floyd-Warshall
Floyd-Warshall(W: матрица на теглата n×n)
1  D ← W              -- D = D^(0)
2  for k ← 1 to n     -- за всеки междинен връх k
3      for i ← 1 to n
4          for j ← 1 to n
5              D[i][j] ← min(D[i][j], D[i][k] + D[k][j])
6  return D           -- D = D^(n) = матрицата на δ(i,j)

Сложност по време: Θ(n³) — три вложени цикъла, всеки до n

Сложност по памет: Θ(n²) — само текущата матрица D (in-place)

Конкретен пример — Попълване на DP матрицата
Floyd-Warshall на граф с 4 върха
Граф (тегла): 1→2: 3, 1→3: 8, 2→3: 1, 2→4: 2, 3→4: -4, 4→2: -1 D^(0) начална матрица: 1 2 3 4 1 [ 0 3 8 ∞ ] 2 [ ∞ 0 1 2 ] 3 [ ∞ ∞ 0 -4 ] 4 [ ∞ -1 ∞ 0 ] k=1 (разглеждаме върха 1 като междинен): D[i][j] = min(D[i][j], D[i][1]+D[1][j]) Няма промени (D[i][1]=∞ за i≠1) D^(1) = D^(0) (без промени) k=2 (разглеждаме върха 2 като междинен): D[1][3] = min(8, D[1][2]+D[2][3]) = min(8, 3+1) = 4 ← ПРОМЯНА D[1][4] = min(∞, D[1][2]+D[2][4]) = min(∞, 3+2) = 5 ← ПРОМЯНА D[4][3] = min(∞, D[4][2]+D[2][3]) = min(∞, -1+1) = 0 ← ПРОМЯНА D^(2): 1 2 3 4 1 [ 0 3 4 5 ] 2 [ ∞ 0 1 2 ] 3 [ ∞ ∞ 0 -4 ] 4 [ ∞ -1 0 0 ] ← забележете D[4][3]=0 k=3 (включваме 3): D[1][4] = min(5, D[1][3]+D[3][4]) = min(5, 4+(-4)) = 0 ← ПРОМЯНА D[2][4] = min(2, D[2][3]+D[3][4]) = min(2, 1+(-4)) = -3 ← ПРОМЯНА D[4][4] = min(0, D[4][3]+D[3][4]) = min(0, 0+(-4)) = -4 ← ОТРИЦАТЕЛЕН ЦИКЪЛ! k=4 (включваме 4): D[1][2] = min(3, D[1][4]+D[4][2]) = min(3, 0+(-1)) = -1 ← ПРОМЯНА ... Финално D^(4): δ(i,j) за всички двойки
Ключови неща за запомняне
  • Floyd-Warshall: Θ(n³) — оптимален за APShP върху гъсти графи
  • Разлика от Shortest-Path-MM: тук броим по вътрешни върхове, там по ребра
  • Проверка за отрицателен цикъл: D[i][i] < 0 за някое i след алгоритъма
  • In-place: не се нуждаем от нова матрица на всяка итерация
  • Може да се разшири за намиране на самите пътища (не само дължините) с матрица на предшествия

Matrix-Chain Multiplication и Неасоциативна операция

3
Изч. Задача 32: Matrix-Chain Multiplication

Дадена е верига от матрици A₁, A₂, ..., Aₙ с размери p₀×p₁, p₁×p₂, ..., pₙ₋₁×pₙ. Намери скобуването, минимизиращо броя скаларни умножения.

Характеризация: Нека m[i][j] = минималният брой умножения за пресмятане на AᵢAᵢ₊₁...Aⱼ.

Рекурсивна декомпозиция:

  • m[i][i] = 0 (базов случай)
  • m[i][j] = mini≤k<j{ m[i][k] + m[k+1][j] + pᵢ₋₁·pₖ·pⱼ }
Алгоритъм 40: Matrix-Chain-Order
-- Алгоритъм 40: Matrix-Chain-Order
Matrix-Chain-Order(p[0..n]: размерите на матриците)
1  for i ← 1 to n
2      m[i][i] ← 0        -- базов случай
3  for len ← 2 to n      -- дължина на подвериги
4      for i ← 1 to n-len+1
5          j ← i + len - 1
6          m[i][j] ← ∞
7          for k ← i to j-1    -- изпробваме всички точки на разрязване
8              q ← m[i][k] + m[k+1][j] + p[i-1]*p[k]*p[j]
9              if q < m[i][j]
10                 m[i][j] ← q
11                 s[i][j] ← k      -- запомняме оптималното k
12 return m, s

Сложност по време: Θ(n³) — три вложени цикъла

Сложност по памет: Θ(n²) — матриците m и s

Пример — 4 матрици с размери 30×35, 35×15, 15×5, 5×10
p = [30, 35, 15, 5, 10] (n=4 матрици) len=1 (базов): m[1][1]=m[2][2]=m[3][3]=m[4][4]=0 len=2 (двойки): m[1][2] = p[0]*p[1]*p[2] = 30*35*15 = 15750 m[2][3] = p[1]*p[2]*p[3] = 35*15*5 = 2625 m[3][4] = p[2]*p[3]*p[4] = 15*5*10 = 750 len=3 (тройки): m[1][3]: k=1: m[1][1]+m[2][3]+p[0]*p[1]*p[3] = 0+2625+30*35*5 = 7875 k=2: m[1][2]+m[3][3]+p[0]*p[2]*p[3] = 15750+0+30*15*5 = 18000 → m[1][3] = 7875, s[1][3] = 1 m[2][4]: k=2: m[2][2]+m[3][4]+p[1]*p[2]*p[4] = 0+750+35*15*10 = 6000 k=3: m[2][3]+m[4][4]+p[1]*p[3]*p[4] = 2625+0+35*5*10 = 4375 → m[2][4] = 4375, s[2][4] = 3 len=4 (четворка): m[1][4]: k=1: m[1][1]+m[2][4]+p[0]*p[1]*p[4] = 0+4375+30*35*10 = 15875 k=2: m[1][2]+m[3][4]+p[0]*p[2]*p[4] = 15750+750+30*15*10 = 21000 k=3: m[1][3]+m[4][4]+p[0]*p[3]*p[4] = 7875+0+30*5*10 = 9375 → m[1][4] = 9375, s[1][4] = 3 Оптимално скобуване: s[1][4]=3 → (A₁A₂A₃)(A₄) → s[1][3]=1 → (A₁)(A₂A₃) = ((A₁(A₂A₃))A₄) с 9375 умножения
Изч. Задача 34: Неасоциативна операция върху стринг

Дадени са символи x₁, x₂, ..., xₙ и неасоциативна двоична операция ⊕. Намери броя на начините да скобуваме израза x₁ ⊕ x₂ ⊕ ... ⊕ xₙ, за да получим даден резултат.

Рекурсивна декомпозиция: Нека ways[i][j][v] = брой скобувания на xᵢ...xⱼ, дали стойност v.

ways[i][j][v] = Σi≤k<j, a⊕b=v{ ways[i][k][a] · ways[k+1][j][b] }

Алгоритъм 41: Non-Associative-Op
-- Алгоритъм 41: Non-Associative-Op
Non-Assoc-Op(x[1..n]: стринг, ⊕: операция, target: цел)
1  for i ← 1 to n
2      ways[i][i][x[i]] ← 1     -- базов: един символ, точно една стойност
3  for len ← 2 to n
4      for i ← 1 to n-len+1
5          j ← i + len - 1
6          for k ← i to j-1
7              for всяка стойност a в ways[i][k]
8                  for всяка стойност b в ways[k+1][j]
9                      ways[i][j][a⊕b] += ways[i][k][a] * ways[k+1][j][b]
10 return ways[1][n][target]

Сложност: Θ(n³ · |V|²), където |V| е размерът на областта на стойностите

Ключови неща за запомняне
  • Matrix-Chain: разрязваме в k → две подзадачи + цената на комбинацията
  • Итерираме по дължина len (1, 2, ..., n) — така подзадачите са решени преди нужда
  • s[i][j] = оптималното k → чрез обратно проследяване намираме скобуването
  • Неасоциативна задача: структурно идентична с Matrix-Chain

2-Partition, Псевдополиномиална сложност и Knapsack

4
Изч. Задача 36: 2-Partition

Дадено е мултимножество от цели числа S = {a₁, ..., aₙ}. Съществува ли разбиване на S на два дяла P и Q с еднакви суми (сумата на P = сумата на Q)?

Рекурсивна декомпозиция: Нека part[i][s] = True, ако съществува подмножество на {a₁,...,aᵢ} с сума s.

  • part[0][0] = True; part[0][s] = False за s > 0
  • part[i][s] = part[i-1][s] OR (s ≥ aᵢ AND part[i-1][s-aᵢ])
  • Отговор: part[n][Sum/2], ако Sum е четно
Алгоритъм 42: 2-Partition DP
-- Алгоритъм 42: 2-Partition
Two-Partition(a[1..n]: цели числа)
1  Sum ← Σa[i]
2  if Sum mod 2 ≠ 0: return False   -- нечетна сума → невъзможно
3  T ← Sum/2
4  part[0..n][0..T] ← False
5  part[0][0] ← True
6  for i ← 1 to n
7      for s ← 0 to T
8          part[i][s] ← part[i-1][s]
9          if s ≥ a[i]
10             part[i][s] ← part[i][s] OR part[i-1][s-a[i]]
11 return part[n][T]

Сложност: Θ(n·Sum) = Θ(n·T)

Псевдополиномиална сложност по време

Алгоритъмът за 2-Partition работи в Θ(n·Sum). Sum е числова стойност от входа, а не размерът на входа. Размерът на входа (в битове) е Θ(n·lg(max aᵢ)).

Алгоритъм е псевдополиномиален, ако сложността му е полином в числовите стойности на входа, но експоненциален в размера на кодирането им. Такива задачи са NP-трудни, но "лесни" при малки числа.

Значение: 2-Partition е NP-пълна, но има псевдополиномиален алгоритъм → "weakly NP-complete".

Изч. Задача 40: Unbounded Knapsack (Неограничена раница)

Дадени са предмети с тегла w₁,...,wₙ и стойности v₁,...,vₙ, и капацитет W. Може да се вземат произволен брой копия от всеки предмет. Максимизирай стойността.

Рекурсивна декомпозиция: Нека uk[c] = максималната стойност за капацитет c.

  • uk[0] = 0
  • uk[c] = maxi: wᵢ≤c{ vᵢ + uk[c - wᵢ] }
Алгоритъм 43: Unbounded-Knapsack
-- Алгоритъм 43: Unbounded Knapsack
Unbounded-Knapsack(w[1..n], v[1..n], W: капацитет)
1  uk[0..W] ← 0
2  for c ← 1 to W
3      for i ← 1 to n
4          if w[i] ≤ c
5              uk[c] ← max(uk[c], v[i] + uk[c - w[i]])
6  return uk[W]

Сложност по време: Θ(n·W) — псевдополиномиална

Сложност по памет: Θ(W)

Изч. Задача 41: 0-1 Knapsack (Ограничена раница)

Всеки предмет може да се вземе най-много веднъж. Нека k[i][c] = максималната стойност, използвайки само предмети от {1,...,i} и капацитет c.

  • k[0][c] = 0; k[i][0] = 0
  • k[i][c] = k[i-1][c], ако wᵢ > c
  • k[i][c] = max(k[i-1][c], vᵢ + k[i-1][c-wᵢ]), ако wᵢ ≤ c
Алгоритъм 44: 0-1-Knapsack
-- Алгоритъм 44: 0-1 Knapsack
01-Knapsack(w[1..n], v[1..n], W: капацитет)
1  k[0..n][0..W] ← 0
2  for i ← 1 to n
3      for c ← 0 to W
4          k[i][c] ← k[i-1][c]      -- не вземаме предмет i
5          if w[i] ≤ c
6              k[i][c] ← max(k[i][c], v[i] + k[i-1][c-w[i]])

Сложност по време: Θ(n·W) — псевдополиномиална

Сложност по памет: Θ(n·W)

Пример — 0-1 Knapsack (4 предмета, W=5)
Предмети: w=[2,3,4,5], v=[3,4,5,8] DP таблица k[i][c]: c: 0 1 2 3 4 5 i=0: [ 0 0 0 0 0 0 ] i=1 (w=2,v=3): [ 0 0 3 3 3 3 ] ↑ ↑ ↑3 ↑3 ↑3 ↑3 (вземаме предмет 1) i=2 (w=3,v=4): [ 0 0 3 4 4 7 ] ↑4 ↑4 ↑7 (вземаме предмет 2) (7=3+4, предмети 1+2) i=3 (w=4,v=5): [ 0 0 3 4 5 7 ] ↑5 (7≥5, не се подобрява) i=4 (w=5,v=8): [ 0 0 3 4 5 8 ] ↑8 (8>7) Отговор: k[4][5] = 8 → вземаме само предмет 4 (w=5, v=8)
Ключови неща за запомняне
  • 0-1 Knapsack: k[i-1][c-wᵢ] (използва предишния ред = предмет i не се повтаря)
  • Unbounded Knapsack: uk[c-wᵢ] (текущия ред = предмет i може да се повтаря)
  • Псевдополиномиална = полином в числовите стойности, но NP-трудна задача
  • И 2-Partition, и Knapsack са NP-трудни → псевдополиномиален алгоритъм = "weakly NP-complete"

Interval Scheduling и Weighted Interval Scheduling

5
Изч. Задача 43: Interval Scheduling (Планиране на интервали)

Дадени са интервали [sᵢ, fᵢ) за i = 1,...,n (начало и край). Намери максимален брой съвместими (непресичащи се) интервали.

Евристика "Най-ранно приключване": Сортираме по f (край) и винаги избираме интервала, приключващ най-рано, стига да е съвместим с вече избраните.

Теорема 107 — Коректност на евристиката за Interval Scheduling

Алгоритъмът "Най-ранно приключване" (earliest finish time) дава оптимален брой съвместими интервали. Доказателство: с "exchange argument" — всяко оптимално решение може да бъде трансформирано в решението на алгоритъма без намаляване на броя.

Алгоритъм 45: Earliest-Finish-Time
-- Алгоритъм 45: Interval Scheduling (алчен алгоритъм)
Earliest-Finish-Time(intervals: списък от (s[i], f[i]))
1  сортирай intervals по f[i] нарастващо   -- Θ(n·lg n)
2  result ← ∅
3  last_finish ← -∞
4  for i ← 1 to n
5      if s[i] ≥ last_finish    -- интервалът е съвместим
6          result ← result ∪ {i}
7          last_finish ← f[i]
8  return result

Сложност: Θ(n·lg n) — доминира сортирането

Изч. Задача 44: Weighted Interval Scheduling

Всеки интервал има тегло vᵢ. Намери подмножество от съвместими интервали с максимална обща стойност.

Рекурсивна декомпозиция: Нека p(i) = индексът на последния интервал, приключващ преди началото на интервал i. Нека OPT[i] = оптималната стойност за интервали {1,...,i}.

  • OPT[0] = 0
  • OPT[i] = max(OPT[i-1], vᵢ + OPT[p(i)])
  • или вземаме интервал i (и добавяме OPT[p(i)]), или не го вземаме (OPT[i-1])
Алгоритъм 46: Weighted-Interval-Scheduling
-- Алгоритъм 46: Weighted Interval Scheduling (DP)
Weighted-IS(s[1..n], f[1..n], v[1..n]: интервали и стойности)
1  сортирай по f[i] нарастващо                    -- Θ(n·lg n)
2  for i ← 1 to n
3      p[i] ← бинарно търсене за последния j: f[j] ≤ s[i]  -- Θ(lg n)
4  OPT[0] ← 0
5  for i ← 1 to n
6      OPT[i] ← max(OPT[i-1], v[i] + OPT[p[i]])
7  return OPT[n]

Сложност по време: Θ(n·lg n) — сортирането и бинарното търсене за p[i]

Сложност по памет: Θ(n)

Конкретен пример — Weighted Interval Scheduling
Пример с 5 интервала
Интервали (сортирани по край): i s f v p[i] 1 1 3 2 0 2 2 5 4 0 3 4 6 4 1 4 6 7 3 2 5 5 9 5 1 OPT[0] = 0 OPT[1] = max(OPT[0], v[1]+OPT[p[1]]) = max(0, 2+OPT[0]) = max(0,2) = 2 OPT[2] = max(OPT[1], v[2]+OPT[p[2]]) = max(2, 4+OPT[0]) = max(2,4) = 4 OPT[3] = max(OPT[2], v[3]+OPT[p[3]]) = max(4, 4+OPT[1]) = max(4, 4+2) = 6 OPT[4] = max(OPT[3], v[4]+OPT[p[4]]) = max(6, 3+OPT[2]) = max(6, 3+4) = 7 OPT[5] = max(OPT[4], v[5]+OPT[p[5]]) = max(7, 5+OPT[1]) = max(7, 5+2) = 7 Отговор: OPT[5] = 7 Обратно проследяване: OPT[5]=7=7=OPT[4] → не вземаме интервал 5 OPT[4]=7≠OPT[3]=6 → вземаме интервал 4 (p[4]=2) OPT[2]=4≠OPT[1]=2 → вземаме интервал 2 (p[2]=0) Оптималното решение: {2, 4} с обща стойност 4+3=7
Ключови неща за запомняне
  • Interval Scheduling без тегла: АЛЧЕН алгоритъм (най-ранен край) → Θ(n·lg n)
  • Weighted Interval Scheduling: ДИНАМИЧНО ПРОГРАМИРАНЕ → Θ(n·lg n)
  • p(i) = последният съвместим интервал преди i → бинарно търсене
  • OPT[i] = max(без i: OPT[i-1], с i: vᵢ + OPT[p(i)])
  • Разликата: тегла правят проблема по-сложен → алчната евристика вече не работи
Връзка с другите теми

Interval Scheduling без тегла е пример за алчен алгоритъм с доказуемо оптимален резултат (Теорема 107). Weighted Interval Scheduling показва, че малка промяна в задачата (добавяне на тегла) може да изисква динамично програмиране вместо алчна схема. Сравнете с МПД: Prim и Kruskal са алчни поради Теорема 93.
